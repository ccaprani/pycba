### Features
- **Collapse-mechanism plot**: `NonlinearResult.plot_collapse()` draws the deflected shape at failure with a large marker at each plastic hinge — the standard plastic-collapse picture. The collapse displacement field is now stored on the result.
- **Rotational springs shown on the schematic**: a vertical support that also carries a rotational spring (e.g. a sub-frame column) is drawn with a spiral, so its moment reaction in `plot_reactions` / `plot_results` is explained rather than appearing unannounced.
- **`plot_vehicle`**: draw a vehicle's axle loads as scaled, labelled arrows along its wheelbase — `cba.plot_vehicle(vehicle)`, also `vehicle.plot()` and `VehicleLibrary.plot_vehicle(vehicle)`. Respects `set_units` for the labels and raises a clear `TypeError` for non-vehicles.
- **`Envelopes.plot_coincidents()`**: plot each load-effect envelope together with the *coincident* value of the other effect, grouped by driving extreme and drawn on **dual y-axes** — the moment envelope with the coincident shear, and the shear envelope with the coincident moment (both as solid lines on their own scale).
- **Shaded result diagrams**: `BeamAnalysis.plot_results` shades the bending-moment and shear-force diagrams with a light fill to the baseline, matching the loading-schematic style — in both the matplotlib and Plotly backends.
- **Consistent "member" naming**: the public API now uses **member** as the primary term for a beam element (the segment between two nodes), reserving *span* for a clear span between supports — which may comprise several members. `Beam.no_members` is added (with `no_spans` kept as a deprecated alias), `add_member` is the documented constructor (`add_span` retained), the load-add methods take `i_member`, and the `Beam`/`BeamAnalysis` reprs report members.
- **Results query & export**: `BeamResults.at(x)` (and the convenience `BeamAnalysis.at(x)`) interpolate the moment, shear, rotation and deflection at any global coordinate; `to_dataframe()` returns the member load effects (`x, M, V, R, D`) as a `pandas.DataFrame` and `to_csv()` writes them (via `numpy`, so no pandas needed). `Envelopes` gains the same `to_dataframe()` / `to_csv()` for the moment/shear envelopes, complementing the existing `Envelopes.at()` and `per_span()`.
- **Reaction plots**: `BeamAnalysis.plot_reactions()` draws the support reactions as labelled arrows on the beam schematic — straight arrows in the force direction (positive upward) for vertical reactions and curved arrows for moment reactions at fixed/encastre supports, with elastic spring-support forces included. The same reactions are added as a final panel of `plot_results` (x-aligned with the moment/shear/deflection diagrams; toggle with `show_reactions=`).
- **Vehicle library reorganised by region, with many more code load models**: `VehicleLibrary` is now grouped into region namespaces — `AU` (AS 5100.2 / NAASRA), `US` (AASHTO / AREA), `EU` (Eurocode EN 1991-2), `UK` (BS 5400 / CS 454), `CA` (CSA S6) and `CN` (JTG D60). New design vehicles, extracted from the national bridge codes: AASHTO HL-93 truck & tandem (`US.get_hl93_truck`, `US.get_hl93_tandem`), the AREA Cooper E-series rail load (`US.get_cooper`, scaled by the E-number), Eurocode Load Model 1 (`EU.get_lm1`) and rail Load Model 71 (`EU.get_lm71`), BS 5400 / CS 454 HB (`UK.get_hb`), CSA CL-625 (`CA.get_cl625`), China JTG D60 standard vehicle (`CN.get_jtg_vehicle`), the AS 5100.2 A160 axle / W80 wheel (`AU.get_a160`, `AU.get_w80`), and the historical NAASRA T44 and MS18 trucks (`AU.get_t44`, `AU.get_ms18`). Each returns a `Vehicle`; the companion lane UDL (HL-93 9.3, LM1 27, LM71 80 kN/m, …) is documented for use as `run_load_model(..., w_lane=…)`.
- **Shear points for moving-load assessment**: request sections at which the shear is recovered *exactly on both sides* (capturing the discontinuity when an axle sits on the section) via `shear_points=` / `dv=` on `BridgeAnalysis.run_vehicle`, `run_load_model` and `static_vehicle`, or directly through `BeamResults(shear_points=...)`. `dv` (the critical-shear distance, e.g. the MCFT effective shear depth) places a section that distance from every vertical support, on each valid side. Sections are placed by a station pair straddling each requested coordinate, so the moving-load evaluation grid stays consistent across the traverse; `BridgeAnalysis.critical_values` gains a `"shear_points"` block reporting the left/right shear envelopes at each section, tagged with its support and side. The helper `resolve_shear_points(beam, points=, dv=)` maps global sections to per-member coordinates. The default evaluation grid is bit-for-bit unchanged when no shear points are requested.
- **Moving load model with lane UDL**: `BridgeAnalysis.run_load_model(step, w_lane, clearances=...)` runs a vehicle together with an accompanying lane UDL, enveloped over the traverse. By default the lane UDL is continuous and runs directly beneath the vehicle (as for the AS5100 M1600, whose 6 kN/m UDL accompanies the truck); a `clearances=(back, front)` clear zone — `back` measured from the rear axle, `front` from the front axle — removes the UDL around the vehicle for models that specify one. This supports code load models pairing a truck with a lane UDL (e.g. AS5100 M1600, AASHTO HL-93, Eurocode LM1). The lane UDL is built from per-span partial UDLs split at span boundaries via the new `BridgeAnalysis._interval_udl_LM` helper.
- **Optional Plotly plotting backend**: interactive, hover-to-read result diagrams and envelopes. `BeamAnalysis.plot_results` / `plot_bmd` / `plot_sfd` / `plot_dsd` and `Envelopes.plot` accept `backend="plotly"` (or set it globally with `pycba.set_backend("plotly")`) and return a `plotly.graph_objects.Figure`; with the shared x-axis of the combined plot, one hover reports the bending moment, shear and deflection at the same section. Plotly is an optional dependency (`pip install pycba[plotly]`); matplotlib remains the default and is unchanged, and the bending-moment panels keep the sagging-positive (reversed-axis) convention.
- **Beam on an elastic (Winkler) foundation**: a span given a finite foundation modulus `kf` (modulus of subgrade reaction per unit length) rests on a Winkler foundation. It is modelled as a statically-condensed beam-on-elastic-foundation super-element — internally meshed into Euler–Bernoulli sub-elements carrying the consistent foundation stiffness, with the internal nodes condensed out — so the two-node assembly, reactions, plotting and influence lines are inherited unchanged, and member diagrams are recovered exactly per sub-element. The mesh defaults to the foundation characteristic length and the result reproduces Hetényi's infinite-beam solution. `kf` is accepted by `Beam`, `Beam.add_span`/`add_member` and `BeamAnalysis` (a scalar for all spans, or one value per span). Supported for prismatic, fixed-fixed spans without `GAv`, carrying UDL / point / partial-UDL loads; other combinations raise a clear error.
- **Free-vibration (modal) analysis**: `BeamAnalysis.modal(mass, n_modes=...)` returns the natural frequencies and mode shapes, assembling a consistent mass matrix alongside the stiffness matrix and solving the generalized eigenproblem `K φ = ω² M φ`. Each span is refined into Euler–Bernoulli sub-elements so the modes are accurate (validated against the classical simply-supported, cantilever and fixed-fixed frequencies to <0.1%). Returns a `ModalResults` (`omega`, `f`, `periods`, mode shapes, `plot_modes()` and a two-panel `plot_results()` showing the beam schematic above the mode shapes); elastic spring supports are included. Prismatic, fixed-fixed spans without `GAv`.
- **Individual result-diagram plots**: `BeamAnalysis.plot_bmd`, `plot_sfd` and `plot_dsd` draw a single bending-moment, shear-force or deflection diagram (the bending moment sagging-positive, with the y-axis inverted, matching `plot_results`). Each accepts an existing `ax`, so two analyses can be overlaid for comparison, and matplotlib `**kwargs` (`color`, `ls`, `label`, …).
- **Timoshenko (shear-deformable) beam elements**: a member becomes a shear-deformable Timoshenko element when given a finite transverse shear rigidity `GAv` (a scalar, or a `SectionEI` for a variable `GAv(x)`); with `GAv=None` (the default) the member stays on the exact Euler–Bernoulli path, bit-for-bit unchanged. Shear deformation flows through the element stiffness, the fixed-end forces and the displacement recovery, for prismatic and non-prismatic members and all four release types, routed through the existing flexibility integrator (no per-load closed forms rewritten). Two DOF per node is preserved, so the `supports=` API, reactions, plotting and influence lines are inherited unchanged. `GAv` is accepted by `Beam`, `Beam.add_span`/`add_member`, `BeamAnalysis` and `BridgeAnalysis.add_bridge`. Nonlinear (plastic-hinge) analysis remains Euler–Bernoulli.

### Breaking changes
- **Load-method parameter `i_span` renamed to `i_member`.** `BeamAnalysis.add_udl`, `add_pl`, `add_pudl`, `add_ml`, `add_ic` and `add_trap` now take `i_member` as the first argument. Positional calls (e.g. `ba.add_udl(1, 10)`) are unaffected; only the rarely-used `i_span=` keyword form changes to `i_member=`. The `no_spans` property and `add_span` method are retained as deprecated aliases.
- **`VehicleLibrary` accessors move under a region namespace.** The flat methods (`VehicleLibrary.get_m1600(…)`, `get_s1600`, `get_la_rail`, `get_abag_bdouble`, `get_example_permit`, …) are now under `VehicleLibrary.AU.*`, e.g. `VehicleLibrary.AU.get_m1600(6.25)` and the new convenience `VehicleLibrary.AU.get_300la(…)`. `get_validation_truck` remains at the top level.

### Fixed
- **Deflected-shape closure verticals**: the result-grid padding that closes the bending-moment and shear diagrams to the baseline was also drawing spurious vertical lines on the deflected shape at member ends (e.g. dropping to zero at a cantilever tip). `plot_results` / `plot_dsd` (matplotlib and Plotly) now use `BeamResults.deflection_curve()`, the continuous de-padded deflection.

## 0.8.0 — 2026-06-11

### Features
- **Nonlinear (elasto-plastic) beam analysis** (#140): `NonlinearBeamAnalysis`, an incremental analysis using the Generalized Clough concentrated-plasticity model. Tracks plastic-hinge formation and moment redistribution to collapse for both proportional static loading and moving vehicle loads, with rank-test mechanism detection. Prismatic members only — a non-prismatic (`SectionEI`) member raises a clear `TypeError`.
- **Non-prismatic (variable-EI) elements** (#139): `SectionEI`, a member whose flexural rigidity varies along its length, built from `const` / `linear` / `pwl` / `poly` segments and analysed exactly by piece-by-piece flexibility integration. Scalar and `SectionEI` members can be mixed in one beam.
- **Imposed-curvature (initial-strain) loads** (#139): load type 6 / `BeamAnalysis.add_ic`, a free curvature field for modelling creep, shrinkage and thermal effects.
- Add coincident load effects to `Envelopes` (#122). New attributes `Vco_Mmax`, `Vco_Mmin`, `Mco_Vmax`, `Mco_Vmin` track the co-existing value of the other effect (V or M) at the truck position that caused each envelope extreme. `critical_values()` output now includes `"Vco"` and `"Mco"` keys. Coincident values are preserved through `augment()` and `zero_like()`.
- Add trapezoidal (linearly varying) distributed load type (#101). Load type 5 supports both full-span `[span, 5, w1, w2]` and partial coverage `[span, 5, w1, w2, a, c]` where w1/w2 are intensities at positions a and a+c respectively. Also adds `BeamAnalysis.add_trap()` convenience method with optional `a` and `c` parameters.
- Add `pos_start` and `pos_end` parameters to `BridgeAnalysis.run_vehicle()` to restrict the vehicle traverse range (#53). Useful for transverse deck analyses where the vehicle is confined to specific lanes.
- Add `Envelopes.sum()` method for element-wise addition of compatible envelopes (#92). This enables superimposing load effects from different sources, e.g. a patterned UDL envelope with a moving vehicle envelope.
- Fix `InfluenceLines.get_il()` raising `IndexError` when the point of interest does not fall exactly on the result grid (#89). This occurred with longer spans where the default 100-point discretization produced a grid spacing that could not match arbitrary poi values. The fix uses per-member result lookup, avoiding both the overly tight floating-point tolerance and the zero-valued padding indices at span boundaries.

### Documentation
- Migrated the documentation to Markdown (MyST); reorganised and expanded the Theoretical Basis page (elements, loads, nonlinear analysis); added tutorials for non-prismatic elements and imposed curvature (#139, #140).
